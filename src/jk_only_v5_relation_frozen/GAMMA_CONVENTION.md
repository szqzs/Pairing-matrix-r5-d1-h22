# Gamma Convention in JK Variables

This folder uses Jeffrey-Kirwan's paper-level odd slant classes

```text
b_r^j = (alpha_j, tau_r(U)),  2 <= r <= 5,  1 <= j <= 4.
```

The symbols `gamma_rs` are not additional generators from the JK paper.  They
are abbreviations in the exterior algebra generated by these paper classes.

We choose the standard symplectic basis of `H_1(Sigma)` ordered as

```text
alpha_1, alpha_2, alpha_3, alpha_4
```

with nonzero pairings

```text
alpha_1 . alpha_3 = 1
alpha_2 . alpha_4 = 1
```

and skew-symmetry giving

```text
alpha_3 . alpha_1 = -1
alpha_4 . alpha_2 = -1.
```

With this ordered symplectic basis, the Sp-invariant abbreviation is

```text
gamma_rs =
    b_r^1 b_s^3 - b_r^3 b_s^1
  + b_r^2 b_s^4 - b_r^4 b_s^2.
```

Because the `b` classes are odd, this is symmetric in `r,s` after exterior
reordering:

```text
gamma_rs = gamma_sr.
```

For example,

```text
gamma_22 = 2 b_2^1 b_2^3 + 2 b_2^2 b_2^4.
```

The code orders exterior variables lexicographically by `(r,j)`:

```text
(2,1),(2,2),(2,3),(2,4),(3,1),...
```

Every product is canonicalized to that order using the Koszul sign for odd
classes.  No rescaling of `b_r^j` is part of this convention.
