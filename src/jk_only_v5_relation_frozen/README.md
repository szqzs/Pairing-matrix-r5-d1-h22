# JK-Only v5 Path

This folder is the paper-first implementation path for the rank-5, genus-2
Jeffrey-Kirwan pairing.

Rules for this folder:

- The only mathematical convention is the one in Jeffrey-Kirwan,
  arXiv:alg-geom/9608029.
- Code here must not import previous pairing implementations.
- Odd classes are the paper slant classes `b_r^j`.
- Any `gamma_rs` is only an abbreviation in the exterior algebra generated by
  those `b_r^j` classes.
- Caches and certificates must record the source formula and code path that
  produced them.

The current files are the foundation for replacing the previous setup kernels:

- `JK_THEOREM_9_6_RANK5_G2.md` records the formula specialization.
- `GAMMA_CONVENTION.md` records the Sp-invariant gamma abbreviation in JK
  `b_r^j` variables.
- `jk_formula.py` implements the paper formula and paper-derived setup
  generators.
- `fast_modular.py` implements a modular fast evaluator backed only by the
  paper-derived formula.  The default kernel compiler is now pure modular
  sparse-polynomial code; the SymPy-backed modular compiler remains as an
  oracle for checks.  Its residue transition uses residue-condition truncation
  bounds instead of a fixed series cutoff, polynomial residues are batched
  through the transition, and column batches reuse shared kernel-gamma factors
  across rows.
- `basis.py` enumerates Sp-invariant products in the JK variables and reduces
  them in the raw exterior algebra on `b_r^j`, without importing any previous
  basis code.
- `residue_oracle.py` runs a slower SymPy-backed modular oracle for selected
  one-variable transitions and composed monomial residues.
- `sample_certificate.py` runs small top-degree JK pairing samples across
  several primes, reconstructs exact rational values, and checks denominator
  avoidance for those samples.
- `speed_probe.py` measures a small modular batch through the JK-only basis
  and evaluator before a long rank run is attempted.
- `modular_rank_search.py` computes JK-only modular pairing columns, caches
  them, orders W26 columns by cheap, probed, or sampled-rank planners, and
  row-reduces incrementally to certify modular rank lower bounds. It can also
  run an adaptive partial-entry search that promotes rank gains from selected
  minor tests without completing every nonselected column. Its cache metadata
  includes formula/code hashes and basis digests, and final rank results record
  selected-minor certificate data.
- `cluster_rank_driver.py` prepares immutable column manifests, computes
  manifest-namespaced atomic per-column shards or task-bundle shards for
  cluster array jobs, plans column waves, and reduces completed shards into
  the same certificate format as the local rank driver.  The reducer validates
  manifest structure, full W-basis task coverage, shard schema, manifest hash,
  source hashes, basis digests, and column-vector hashes before using any
  shard.  Reduce outputs are structured certificate artifacts and can be
  independently checked with `verify-certificate`.
- `strict_degree_runner.py` is the publication-run wrapper. It writes a
  timestamped per-degree ledger/provenance record, expands every artifact path
  absolutely, runs task-bundle batches instead of `--all-tasks`, reduces and
  verifies after each batch, and publishes only after a final second-prime
  certificate check.
- `theorem_assisted_c12_candidate.py` extracts the current theorem-assisted
  c12 candidate line from loaded c12 shards.  It deliberately does not claim a
  full-W relation certificate.
- `run_checks.py` runs structural and guardrail checks for this folder.

The default checks intentionally avoid large rank computations. They verify the
formula layer, gamma expansion, provenance guardrails, mixed/higher-delta
agreement between `fast_modular.py` and the symbolic JK-only generators,
batched-vs-termwise residues on real pairing polynomials, cache invalidation,
checkpoint certificate semantics, and a tiny cluster manifest/worker/reduce
roundtrip. They do not certify the large `c=13,...,16` computations by
themselves.

Run the current certificate checks with:

```bash
python sp_invariant_fast_algorithm_v5/jk_only/run_checks.py
```

Probe the ordinary-degree 22 source slices and W26 test basis with:

```bash
python sp_invariant_fast_algorithm_v5/jk_only/basis.py
```

Run a small speed probe with:

```bash
python sp_invariant_fast_algorithm_v5/jk_only/speed_probe.py
```

Run the modular rank-search driver with a small cap first:

```bash
python sp_invariant_fast_algorithm_v5/jk_only/modular_rank_search.py --chern-degrees 22 --max-new-columns 3
```

For a sampled actual-pairing planner, use:

```bash
python sp_invariant_fast_algorithm_v5/jk_only/modular_rank_search.py --chern-degrees 14 --column-order cheap-probe --max-new-columns 10
```

For a sampled-rank planner, use:

```bash
python sp_invariant_fast_algorithm_v5/jk_only/modular_rank_search.py --chern-degrees 14 --column-order rank-sampled --rank-planner-pool-size 80 --max-new-columns 10
```

For positive-certificate searches that should avoid completing every
nonselected column, use adaptive partial evaluation:

```bash
python sp_invariant_fast_algorithm_v5/jk_only/modular_rank_search.py --chern-degrees 18 --column-order cheap-probe --evaluation-mode adaptive-partial
```

For publication-grade local runs, prefer the strict wrapper:

```bash
RUN_ROOT="/absolute/path/to/jk_v5_runs/$(date -u +%Y%m%dT%H%M%SZ)"
python sp_invariant_fast_algorithm_v5/jk_only/strict_degree_runner.py run-degree \
  --chern-degree 18 \
  --run-root "$RUN_ROOT" \
  --column-order cheap-probe \
  --columns-per-task 16 \
  --wave-size 64 \
  --task-batch-size 4 \
  --second-prime 1000033
```

For cluster runs, use `cluster_rank_driver.py` directly, but pass absolute
`--manifest`, `--shard-dir`, and `--output` paths every time. For local smoke
tests only, `worker --all-tasks --wave-index 0` computes an entire wave in one
process. For production runs, prefer `wave` to get the exact SLURM array range
for the next unresolved wave, then launch array workers or small explicit task
ranges with `--no-repair-existing`.

For Yale-scale runs, prefer `--shard-mode task` with `--columns-per-task 8` or
`16`. The strict local wrapper defaults to `16` to reduce subprocess and
filesystem overhead. The old per-column shard mode is kept for debugging and
exact single-column reruns.

When using `wave --previous-reduce`, also provide the matching
`--previous-verification` artifact. This prevents wave planning from trusting a
reduce file that has not passed selected-minor recomputation. If
`verify-certificate --second-prime` is supplied, the second-prime determinant
must also be nonzero.

The recommended staged reproduction order for ordinary degree 22 is `c18`,
`c17`, then `c16,c15,c13,c14`, then `c19,c20,c21,c22`, and finally `c12,c11`.
This order is operational only; each Chern degree has its own manifest and
certificate.
